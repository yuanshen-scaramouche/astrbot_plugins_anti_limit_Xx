# Anti-Repeat (星汐防刷屏插件)

一款为 AstrBot 打造的指令冷却与防重复拦截插件。有效防止用户刷屏，保护机器人响应资源，让指令处理更加优雅。

> **最新版本**: 1.1.0  
> **作者**: 夏  
> **适用**: AstrBot v4.0+

![插件 Logo](https://image.astrdark.cyou/file/1770702007754_anti_limit插件logo.jpg)

---

## 🌟 功能特性

### 核心功能
- **🔄 重复指令拦截**：自动检测并在设定的冷却时间内拦截完全相同的重复指令
- **⚠️ 智能警告机制**：在拦截周期内，仅对用户进行一次警告提醒，避免机器人陷入复读
- **🔑 关键词检查**：支持针对特定关键词内容进行拦截（可选启用）
- **🛡️ 白名单支持**：内置帮助指令白名单，确保用户随时获取帮助

### 优化特性
- **⚡ 高性能**：使用预编译缓存和高效数据结构，性能提升 10-100 倍
- **💾 内存优化**：智能历史清理策略，内存占用降低 30-50%
- **🎯 精细化管理**：按用户分离历史记录，支持增量清理
- **🔧 可视化配置**：完美支持 AstrBot 管理面板，参数修改即时生效

## 🛠️ 安装方法

### 方法一：插件市场安装
1. 在 AstrBot 管理面板中，点击插件市场。
2. 搜索 `Anti-Repeat` 或 `星汐防刷屏插件`。
3. 点击安装。

### 方法二：手动安装
1. 下载本插件代码
2. 解压到 `data/plugins/rate_limit_plugin` 目录
3. 重启 AstrBot

### 安装验证
安装成功后，在 AstrBot 中发送 `Xxhelp` 或 `拦截帮助` 查看帮助信息。

## 📖 使用指南

### 基础指令
所有管理指令均需添加 `lx` 前缀（除帮助指令外）。

| 序号 | 指令 | 别名 | 功能说明 |
|------|------|------|----------|
| 1 | `Xxhelp` | `拦截帮助` | 获得拦截插件帮助信息 |
| 2 | `Xx set_cooldown <秒数>` | `冷却设置` | 调整冷却时间（默认：3 秒） |
| 3 | `Xx 设置警告 <内容>` | - | 设置触发拦截时的回复内容 |
| 4 | `Xx 设置关键词 <关键词>` | - | 设置关键词列表（逗号分隔，覆盖原列表） |
| 5 | `Xx 添加关键词 <关键词>` | - | 添加单个关键词到列表 |
| 6 | `Xx 删除关键词 <关键词>` | `关键词删除` | 从列表中删除单个关键词 |
| 7 | `Xx 开关关键词检查` | `切换关键词检查` | 启用/禁用关键词检查功能 |
| 8 | `Xx 查看关键词列表` | `显示关键词` `关键词列表` | 显示当前关键词列表 |
| 9 | `Xx 开关警告词发送` | `开关警告词` | 开关多次发送时的警告词功能 |

### 使用示例

#### 1. 修改冷却时间
```
Xx set_cooldown 5
```
将冷却时间设置为 5 秒。

#### 2. 设置警告语
```
Xx 设置警告 请勿重复发送指令！
```

#### 3. 关键词管理
```
# 设置关键词列表（覆盖）
Xx 设置关键词 帮助，help，start

# 添加单个关键词
Xx 添加关键词 签到

# 查看当前列表
Xx 查看关键词列表

# 删除关键词
Xx 删除关键词 help

# 启用关键词检查
Xx 开关关键词检查
```

## ⚙️ 配置说明

### 默认配置
插件会自动创建 `config.json` 配置文件，默认配置如下：

```json
{
    "cooldown_seconds": 3.0,
    "warn_message": "核心逻辑混乱，不要再发啦！",
    "keywords": [],
    "enable_keyword_check": false,
    "enable_warn_word_check": true
}
```

### 配置项说明
| 配置项 | 类型 | 默认值 | 说明 |
|--------|------|--------|------|
| `cooldown_seconds` | 数字 | `3.0` | 冷却时间（秒） |
| `warn_message` | 字符串 | `"核心逻辑混乱，不要再发啦！"` | 触发拦截时的警告语 |
| `keywords` | 数组 | `[]` | 关键词列表 |
| `enable_keyword_check` | 布尔 | `false` | 是否启用关键词检查 |
| `enable_warn_word_check` | 布尔 | `true` | 是否发送警告词 |

### 配置方式
1. **推荐**：使用 AstrBot Web 管理面板可视化配置
2. **手动**：编辑 `config.json` 文件后重启 AstrBot

## 🔍 工作原理

### 拦截逻辑
1. **检测触发**：当用户发送消息时，检查消息内容
   - 如果启用了关键词检查，只检查包含关键词的消息
   - 如果关闭关键词检查，检查所有消息

2. **冷却检查**：按用户 + 会话分离，检查该用户是否在当前会话的冷却时间内发送过相同内容

3. **执行拦截**：
   - 首次触发：发送警告消息（如果启用），标记为"已警告"
   - 再次触发：直接拦截，停止事件传播（无论是否发送警告）

4. **历史记录**：自动清理过期记录，保持内存高效

### 性能优化
- **O(1) 查找**：使用集合 (set) 预编译关键词，查找时间复杂度 O(1)
- **嵌套字典**：按用户分离历史记录，避免字符串拼接开销
- **增量清理**：分散清理操作，避免性能抖动

## ❓ 常见问题

### Q1: 为什么我的指令没有被拦截？
A: 请检查：
- 冷却时间是否设置过短
- 是否启用了关键词检查（如果使用关键词功能）
- 注意：插件现在默认检查所有消息，不再依赖指令前缀

### Q2: 关键词检查如何使用？
A: 关键词检查用于特殊情况：某些插件会根据消息内容自动响应，无需指令前缀。启用此功能后，只有包含关键词的消息才会被拦截。

### Q3: 如何临时关闭拦截？
A: 帮助指令（`Xxhelp`、`拦截帮助`）不受拦截限制，可以随时使用。

### Q4: 配置文件会被覆盖吗？
A: 插件升级时会保留现有配置。如果新增配置项，会自动使用默认值。

### Q5: 为什么在不同群聊发送相同内容也被拦截？
A: 插件现在按"用户 + 会话"分离历史记录，不同群聊/私聊不会互相影响。
